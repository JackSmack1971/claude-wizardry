# Reality-Grounded Engineering

Use this when making or reviewing software changes.

## Principles

- Claims are not verdicts.
- Passing tests are evidence, not proof of production behavior.
- Logs are observations, not authoritative state.
- Read source-of-truth state before and after important changes.
- Preserve uncertainty until evidence closes it.

## Report Format

- Claim
- Evidence
- Source of truth checked
- Verification result
- Remaining uncertainty
